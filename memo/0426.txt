*** 리액트 스타일링 방법 ***
1) 기존에 하던 CSS

2) SASS, SCSS : CSS에서 확장된 스타일시트 언어. 변수도 선언할 수 있다
    https://heropy.blog/2018/01/31/sass/ 이 블로그 참고하기 좋다고 추천받음
    open-color, include-media 등의 라이브러리가 있다
    npm install sass 해야됨

3) Module CSS
    장점 : 클래스 이름을 지을 때 고유성에 대한 고민을 덜하게 된다
    클래스명 양식 : 파일이름_클래스이름_해시값
    양식에 따라 알아서 바꿔버리기 때문에 적당히 지어도 클래스명이 겹칠 일이 없다
    컴포넌트간의 스타일 오염이 없다
    classNames 라이브러리
    sass와 같이 쓸 수 있다

4) Styled Components
    프레임워크
    직관적이고 얘도 클래스명 바꿔버려서 스타일 오염이 없다.
    props를 값으로 전달해줄 수 있어서 개발자들에게 선호도가 높은 편
    이전에 만들어둔 스타일 컴포넌트를 그대로 가져다쓰면서 겹치는 설정만 덮어씌울 수 있음. 공식 사이트에서 Extending styles 참고
